# Prompt Library — Opportunity Arbitrage

## 1. Money-Trail Scan
```text
You are a ruthless opportunity evaluator.

Find evidence of current spending, expensive mistakes, recurring frustration, or ugly manual workflows in [NICHE / COMMUNITY].

Look specifically for:
- tools or services people already pay for
- complaints about those tools or services
- statements with dollar amounts
- mistakes that cost money, time, or reputation
- repeated workflow pain
- direct quotes like “I would pay for…”, “I hate using…”, “I spent…”, “I wish there was…”

Return:
1. buyer type
2. exact pain pattern
3. current workaround
4. proof of spend or proof of pain
5. possible wedge
6. why this may or may not be a real opportunity

Do not generate startup ideas without evidence.
Kill weak signals early.
```

## 2. Public-Data Arbitrage Scan
```text
Evaluate this public data source as a product opportunity.

For the dataset below, determine:
1. who would care about this data
2. what decision, alert, or search query it could power
3. whether the current interface is weak
4. whether there is a trigger-based commercial use case
5. whether it fits one of these shapes:
   - searchable intelligence
   - alerting product
   - lead-gen intelligence
   - pSEO / long-tail content product
   - compliance monitor

Then score it on:
- buyer clarity
- pain severity
- monetization clarity
- build feasibility
- defensibility
- distribution realism

Dataset: [DATASET / URL / DESCRIPTION]
```

## 3. Wedge Finder
```text
Take this broad market or product idea and reduce it to the strongest narrow wedge.

For the idea below, identify:
1. the clearest buyer
2. the most painful narrow job
3. the strongest trigger moment
4. the smallest sellable first version
5. what to cut from v1

Prefer narrow, painful, monetizable wedges over broad category ideas.

Idea: [IDEA]
```

## 4. Validation Sprint Planner
```text
Create a 30-day validation sprint for this opportunity.

Include:
- week 1: evidence gathering
- week 2: offer testing
- week 3: manual delivery
- week 4: build / backlog / kill decision

For each week, specify:
- objective
- tasks
- output
- success signal
- failure signal

Opportunity: [OPPORTUNITY]
```

## 5. Brutal Critic
```text
Act like a skeptical operator reviewing this opportunity.

Tell me:
1. why the buyer may not care enough
2. where the monetization story is weak
3. what makes the build path heavier than it looks
4. why distribution may be harder than assumed
5. what narrower wedge I should test instead

Be direct. Kill it if the evidence is weak.

Opportunity: [OPPORTUNITY]
```
