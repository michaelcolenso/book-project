# Opportunity Radar MVP

## Objective

Build a repeatable system that surfaces high-signal, monetizable opportunities from public data, communities, and operational pain.

## V0 scope

Manual-first.
No scraping required to be useful.

### Inputs
- manually curated opportunity candidates
- short evidence notes for each score
- optional source links

### Processing
- weighted score calculation
- ranking
- markdown report generation

### Outputs
- ranked opportunities
- strongest wedge recommendation
- visible weak points / disqualifiers

## Why start manual-first

Automation makes bad taste faster.
We want the framework right before we automate ingestion.

## V1 source adapters

### Public-data sources
- awesome-public-datasets
- state / county / city open-data catalogs
- ArcGIS / ESRI endpoints
- public notices / permits / inspections / code enforcement

### Demand / pain sources
- Reddit
- Google Trends
- G2 / Capterra / app reviews
- Hacker News / Indie Hackers
- niche forums and Facebook groups

## Candidate patterns to prioritize

- monitoring / alerts
- compliance / risk visibility
- pricing / inventory / permit intelligence
- lead-generation tools tied to operational pain
- tools that turn ugly public data into cleaner decisions

## Anti-goals

- broad consumer apps with vague monetization
- enterprise workflows requiring long sales cycles
- products that depend on unreliable private APIs for core value
- ideas with unclear buyer and unclear route to distribution
