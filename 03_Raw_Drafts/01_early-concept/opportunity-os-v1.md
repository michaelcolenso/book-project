# Mike's Opportunity OS v1

## Core Rule
Do not chase interesting. Chase painful, triggered, reachable, and shippable.

## The 3 Opportunity Buckets

### 1. High-Stakes Knowledge Products
Use when:
- Buyer is in a first-time, stressful situation
- Information is scattered
- Mistakes are expensive
- Search intent is urgent
- Buyer wants certainty, not software

Best fits:
- Executor playbook
- Insurance claim maximizer
- Property tax appeal playbook
- Renovation self-defense guide
- Medical bill negotiation kit

### 2. Workflow / Compliance SaaS
Use when:
- Pain recurs weekly or monthly
- Workflow is messy, repetitive, or rules-heavy
- User already uses spreadsheets, email, or bad tools
- ROI is operationally obvious
- Retention is plausible

Best fits:
- Permit navigator
- Reseller pricing / inventory / tax tool
- Etsy compliance checker
- Bookkeeper document collection
- HOA board ops
- Hot sauce compliance navigator

### 3. AI Decision / Identification Tools
Use when:
- AI can compress a painful manual process into a fast answer
- Demo value is high
- Buyer sees value in minutes
- Accuracy can be “useful enough” without perfection

Best fits:
- Estate inventory + pricing
- Genealogy document decoder
- Reseller value lookup
- Supplier verification / comparison
- Equipment recommendation engines

## Hard Filters
Every opportunity must answer all 7:
1. Who is the buyer?
2. What exact event or recurring pain triggers action?
3. What does the buyer currently spend: money, time, or errors?
4. How do we reach the first 10 users?
5. Can an ugly but useful v1 ship in 14–21 days?
6. Is there a narrow wedge before expansion?
7. Does this fit Mike’s strengths in systems, ops, data, and workflow design?

If 2+ answers are fuzzy, downgrade or kill.

## Kill Criteria
Kill ideas that are:
- Interesting but not urgent
- Painful but not budgeted
- Technically cool but distribution-vague
- Broad marketplaces too early
- Multi-jurisdiction data monsters without a narrow launch plan
- Products where trust/legal burden is too high for a solo v1
- “Would be nice” tools with no trigger moment

## Scoring Rubric
Score 1–5 on each.

### Universal
- Buyer clarity
- Pain severity
- Willingness to pay / existing spend
- Distribution access
- Speed to ugly MVP
- Defensibility / data moat
- Founder fit

### Add for Knowledge Products
- Urgency of search
- Cost of mistake avoided
- Variant expansion potential (state / scenario / industry)

### Add for Workflow SaaS
- Frequency of use
- Workflow centrality
- Retention likelihood
- Switching pain / stickiness

### Add for AI Decision Tools
- Demo wow factor
- Time-to-value
- Tolerance for imperfect accuracy
- Data flywheel potential

## Decision Tree

### Choose Knowledge Product if:
- Problem is first-time + high-stakes
- User wants instructions, scripts, checklists, templates
- Search behavior is explicit
- Recurring software use is unlikely

### Choose Workflow SaaS if:
- Problem happens repeatedly
- User already has a workaround stack
- ROI is measurable
- Ongoing tracking / reminders / records matter

### Choose AI Decision Tool if:
- A hard, manual task can be compressed into a near-instant result
- A strong demo can sell the product
- The product can be “good enough” before perfect

## Operating Rules
- Recommendation first, reasoning second, next move last.
- Narrow hard.
- Build the wedge, not the empire.
- Validate with workflow and buyer behavior, not compliments.
- Prefer products that produce reusable assets, data, and distribution leverage.
